## 输入格式

输入为给定坐标系下的波形：

h(t<sub>n</sub>) = h<sub>+</sub>(t<sub>n</sub>) - i h<sub>×</sub>(t<sub>n</sub>).

输入文件必须是 CSV 文件，其中每一个复数样本会被解释为一个等间隔采样点。播放器不读取时间网格。

每个可解析的 CSV 单元格都可以是一个复数样本，例如：

```csv
0.0012+0.0004j
0.0011+0.0005j
0.0009+0.0007j
```

也可以使用 `i` 作为虚数单位：

```csv
0.0012+0.0004i
0.0011+0.0005i
0.0009+0.0007i
```

解析规则：

- 空行会被忽略。
- 以 `#` 开头的行会被当作注释忽略。
- 表头或其他无法解析为复数的单元格会被跳过。
- 如果第一列是整数索引，而后面的列包含复数，第一列会被当作行号跳过。
- 如果一个单元格只有实部，例如 `0.0012`，它会被解释为虚部为 0 的复数。
- 文件至少需要包含 16 个可解析的复数样本。
- 当前版本默认所有样本是等间隔采样，不接受单独的时间列。
- 如果 CSV 来自 pandas 导出并包含第一列整数行号，播放器会自动忽略这一列。

## 从复数波形到两个声部

CSV 中的每个复数样本记为 h<sub>n</sub>。程序先还原两个极化分量：

- h<sub>+</sub>(n) 是 h<sub>n</sub> 的实部。
- h<sub>×</sub>(n) 是 h<sub>n</sub> 虚部的相反数。

然后生成两条音乐轨道：

- `h_plus`：由 h<sub>+</sub> 生成。
- `h_cross`：由 h<sub>×</sub> 生成。

## 音高

音高来自相位。

程序先计算每个复数样本的相位。相位记录在 -π 到 π 之间，程序会把穿过边界时的绕回补回来，让相位曲线连续，然后用相邻采样点的相位差估计一个“瞬时频率”。因为输入没有时间网格，这个频率的单位不是 Hz。

接着程序对这个频率做对数归一化，并映射到 MIDI 音高范围 42 到 84。`h_cross` 轨道会整体上移 7 个半音，让两个极化声部在听感上分开。因此它的实际音高可能高于 84。

## 力度

力度来自振幅。

对 `h_plus`，力度来自 h<sub>+</sub> 的绝对值。对 `h_cross`，力度来自 h<sub>×</sub> 的绝对值。程序会把幅度归一化到音乐力度范围内。

## 时间与音符密度

当前版本会把整段输入波形压缩或展开到 90 秒的音乐时间中。

每条声部的目标音符数量由样本数自动决定：

targetNotes = clamp(12 sqrt(N), 140, 720)

其中 N 是复数样本数量。由于会生成 `h_plus` 和 `h_cross` 两条声部，全曲音符总数通常接近两条声部之和。相邻采样区间会被分组。每一组生成一个音符：

- 音符开始时间来自这一组在全曲中的相对位置。
- 音符时长来自这一组覆盖的采样范围。
- 音符音高来自这一组内的频率。
- 音符力度来自这一组内的极化幅度。

## 输出结果

当前版本不会导出 `.mid` 文件，它会把 CSV 转换成内部音符，并载入播放器用于播放和可视化。
